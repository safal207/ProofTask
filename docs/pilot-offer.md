# ProofTask Pilot Offer

## Offer name

AI Agent QA Proof Audit

## One-line offer

ProofTask helps AI-heavy software teams verify agent-generated work with structured human proof before merge, release, or payment.

## Buyer problem

AI coding agents can create features, tests, fixes, and pull requests quickly.

But speed creates a new quality problem:

> Teams need proof that agent-generated work was checked by a human in a real environment.

Current verification evidence is usually scattered across:

- pull request comments;
- screenshots;
- chat messages;
- QA notes;
- bug trackers;
- local test runs;
- informal reviewer judgement.

That makes it hard to answer later:

- what exactly was checked;
- who checked it;
- what proof was submitted;
- why it was accepted or rejected;
- whether the work was safe to merge or release.

## Target customer

Best early customers:

- small SaaS teams using AI coding tools;
- founders shipping with Cursor, Copilot, Codex, Claude Code, or similar tools;
- devtool teams building AI-agent workflows;
- QA agencies creating AI-native services;
- open-source maintainers receiving AI-generated pull requests;
- regulated teams that need audit evidence before release.

## Pilot scope

The first pilot should verify one concrete AI-generated change.

Examples:

- AI agent changed signup flow;
- AI agent generated a pull request;
- AI agent fixed a UI bug;
- AI agent changed checkout, onboarding, billing, or account settings;
- AI agent updated a workflow that needs human confirmation.

## Pilot workflow

1. Select one AI-generated feature, pull request, or workflow.
2. Define acceptance criteria.
3. Create a structured ProofTask verification task.
4. Human tester performs the check in a real browser or device.
5. Human submits required proof.
6. Verifier accepts or rejects the proof.
7. ProofTask produces a traceable verification artifact.
8. Team receives a release recommendation.

## Deliverables

The pilot delivers:

- structured verification task;
- explicit acceptance criteria;
- proof requirements;
- human proof package;
- verification decision;
- traceable audit artifact;
- short findings report;
- release recommendation;
- suggested next integration step.

## What this is not

This is not:

- a full QA outsourcing contract;
- a bug bounty program;
- a generic freelance marketplace;
- a replacement for automated tests;
- a complete compliance system.

This is a focused proof workflow for agent-created work.
