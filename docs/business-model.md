# ProofTask Business Model

## Monetization thesis

ProofTask should not start as a general marketplace.

The first business model should be pilot-led:

> Sell evidence-backed verification workflows to teams already using AI coding agents.

The product should make verification visible, repeatable, and auditable.

## Phase 1: Paid pilots

The fastest path to revenue is not a self-serve SaaS plan.

The fastest path is to sell focused pilots to teams that already use AI coding agents and feel review or QA pressure.

### 1. Agent QA Proof Audit

Suggested price range: `$2,500-$5,000`

A focused audit of one AI-generated feature, pull request, or workflow.

Deliverables:

- structured verification task;
- human proof package;
- traceable verification artifact;
- findings report;
- recommended release decision.

Best for:

- teams experimenting with AI-generated pull requests;
- founders using AI coding tools;
- small SaaS teams without mature QA process.

### 2. ProofTask Integration Pilot

Suggested price range: `$7,500-$15,000`

A lightweight integration of ProofTask into a team's existing development workflow.

Deliverables:

- GitHub-based task creation flow;
- proof templates;
- trace export;
- basic reviewer workflow;
- pilot report;
- integration roadmap.

Best for:

- teams using Cursor, Copilot, Codex, Claude Code, or similar tools;
- AI coding agent builders;
- QA agencies building AI-native services.

### 3. AI Release Gate Pilot

Suggested price range: `$20,000-$50,000`

A stronger pilot for teams that want ProofTask as a release gate before merging or deploying agent-created changes.

Deliverables:

- agent-created task flow;
- human QA proof flow;
- verifier decision flow;
- audit log;
- release recommendation;
- team workflow documentation;
- integration roadmap.

Best for:

- fintech;
- healthtech;
- infrastructure tools;
- regulated teams;
- teams where release mistakes are expensive.

## Phase 2: SaaS

Once the repeated pilot workflow is clear, ProofTask can become a SaaS product.

Potential pricing:

| Plan | Price range | Target user |
| --- | ---: | --- |
| Open-source CLI | Free | Developers and early adopters |
| Team | `$99-$299/month` | Small AI-heavy dev teams |
| Pro | `$499-$1,500/month` | Teams with recurring QA verification |
| Enterprise | `$2,000-$10,000+/month` | Audit-heavy or regulated teams |

Potential paid features:

- hosted dashboard;
- GitHub integration;
- task templates;
- proof storage;
- trace search;
- reviewer roles;
- audit export;
- SSO;
- private deployment;
- compliance reporting;
- API access for agents.

## Phase 3: Verification marketplace

A marketplace should come later, after buyer demand is proven.

Potential revenue streams:

- take rate on completed verification tasks;
- payment release fee;
- reviewer reputation system;
- verified QA worker network;
- enterprise-managed reviewer pool.

The marketplace is attractive, but it is operationally heavy.

ProofTask should not build supply before proving demand.

## Buyer pain

The buyer is not paying for another task tracker.

The buyer is paying for confidence:

- confidence that agent-generated work was checked;
- confidence that verification evidence exists;
- confidence that release decisions are auditable;
- confidence that human review is not lost in comments and screenshots;
- confidence that AI speed does not destroy quality.

## Sales message

Simple sales message:

> Your AI agents can ship faster. ProofTask makes sure humans can verify what matters before you merge, release, or pay.

## Strategic rule

Do not build a marketplace before proving buyer demand.

First sell:

> Verified proof for agent-created work.

Then productize what repeats.
