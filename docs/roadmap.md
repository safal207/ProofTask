# ProofTask Roadmap

ProofTask is intentionally starting small.

The first goal is not to build a marketplace. The first goal is to make one unit of agent-requested human verification traceable.

## Sprint 0.1 — Traceable verification primitive

Status: in progress

Goal:

> Agent task + human proof + verifier decision = traceable verification artifact.

Implemented:

- Task schema
- Proof schema
- Trace schema
- Example task and proof fixtures
- CLI validation for task, proof, and trace
- Submitted trace creation
- Verified/rejected trace creation
- Negative fixtures
- CI smoke test
- Pytest regression tests
- Installable local CLI via `pyproject.toml`

Remaining:

- Decide whether to commit generated example traces or keep them generated-only
- Add JSON Schema validation as an optional stricter check
- Add more rejection scenarios

## Sprint 0.2 — Agent-facing interface

Goal:

> Make it easy for an AI coding agent or toolchain to create a ProofTask task.

Possible scope:

- `prooftask create-task` command
- task templates for manual QA
- task IDs generated by CLI
- markdown-to-task conversion
- first simple agent adapter spec

## Sprint 0.3 — Local task ledger

Goal:

> Store tasks and traces in a small local append-only ledger.

Possible scope:

- `tasks/` local directory layout
- append-only event log
- export final trace
- list tasks by status
- inspect task lifecycle

## Sprint 0.4 — Minimal web UI

Goal:

> Show agent-created tasks and submitted proof in a simple browser interface.

Possible scope:

- task list
- task detail
- proof submission form
- verification decision form
- trace viewer

## Sprint 0.5 — Payment placeholder

Goal:

> Represent payment state safely without processing real money yet.

Possible scope:

- payment intent object
- escrow-like state machine
- manual release/refund status
- audit events for payment changes

## Long-term direction

ProofTask can grow into:

- human QA verification for AI coding agents
- agent-to-human microtask delegation
- compliance evidence workflows
- real-world confirmation tasks
- verification marketplace infrastructure
- proof and trace layer for agentic work

The long-term thesis:

> As AI agents do more work, human proof becomes a critical trust primitive.
